# ⭐ Top 3 — Worth Your Time

## 1. Kafka Tiered Storage in Production

* **Source:** Confluent Blog
* **Link:** [https://www.confluent.io/blog/kafka-tiered-storage/](https://www.confluent.io/blog/kafka-tiered-storage/)

**Summary:**

* **Problem:** Kafka retention is limited by broker disk → expensive scaling
* **Key idea:** Offload older segments to object storage (S3), keep hot data local
* **Why it matters:**
  • Decouples compute vs storage (very similar to lakehouse pattern)
  • Enables long retention without huge broker cost
  • Critical for event sourcing + replay-heavy systems

## 2. Data Skipping in Lakehouse (Databricks)

* **Source:** Databricks Blog
* **Link:** [https://www.databricks.com/blog/2023/06/15/data-skipping-indexing.html](https://www.databricks.com/blog/2023/06/15/data-skipping-indexing.html)

**Summary:**

* **Problem:** Full table scans in data lakes kill performance
* **Key idea:** Use file-level statistics (min/max, bloom filters) to skip files
* **Why it matters:**
  • Acts like “indexing without indexes”
  • Massive I/O reduction in Spark jobs
  • Essential for Iceberg/Delta scale
